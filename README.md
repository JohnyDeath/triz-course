# ТРИЗ-курс для архитектора систем учёта

Одностраничный сайт для прохождения 9-недельного курса по ТРИЗ
(Теория решения изобретательских задач), адаптированного под архитекторов
и разработчиков автоматизированных систем учёта.

**Открыть курс:** https://triz-course.martynenkov.pro/

## Что внутри

- 9 недель × 7 дней (всего 63 занятия), ~30–40 минут в день.
- Для каждого дня: цель, материалы со ссылками, опорный конспект, практика
  и шаблон фиксации в журнале.
- Поле для собственных конспектов с автосохранением в `localStorage` браузера.
- Отметка прохождения и автоматический переход к следующему дню.
- Экспорт всех конспектов в Markdown одной кнопкой.
- Тёмная/светлая тема (автоматически по системной настройке).
- Адаптивный интерфейс: сайдбар с навигацией по неделям превращается
  в выезжающее меню на мобильных устройствах.
- Полностью статический сайт, без серверной части. Работает офлайн
  после первой загрузки.

## Хранение данных

Все ваши конспекты и прогресс сохраняются в `localStorage` браузера.
Это значит:

- На разных устройствах состояние своё.
- Очистка кеша браузера или режим инкогнито сбросят данные.
- Чтобы перенести данные между устройствами — используйте кнопку
  «⬇ Экспорт конспектов (.md)» в сайдбаре.

## Локальный запуск

Достаточно открыть `index.html` двойным кликом —
все зависимости (CSS, JS, данные курса) встроены в файл.

Если хочется запустить через локальный сервер:

```bash
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

## Структура

```
.
├── index.html      # сам сайт (всё в одном файле)
├── CNAME           # привязка к собственному домену для GitHub Pages
├── .nojekyll       # отключает Jekyll-обработку на GitHub Pages
└── README.md       # этот файл
```
